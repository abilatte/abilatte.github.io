## 编译 - GCC

### 安装

>   `gcc`, `g++`是 GNU 的 c 和 c++ 编译器

```shell
# 安装 gcc 和 g++
$ sudo apt install gcc g++
# 查看版本
$ gcc -v
$ g++ -v
```



### 工作流程

>   gcc 对程序的编译分为四个阶段, 预处理(预编译), 编译, 汇编, 链接. gcc 可以将四个阶段合并成一个. 先介绍每个步骤做了什么

1.   预处理 : 这个阶段做三件事 : **展开头文件**, **宏替换**, **去掉注释**
2.   编译 : 对文件进行编译, 得到一个汇编文件
3.   汇编 : 对文件进行汇编, 得到一个二进制文件
4.   链接 : 对程序需要的库进行链接, 得到可执行文件

| 文件后缀 |        说明        | gcc 参数 |
| :------: | :----------------: | :------: |
|   `.c`   |       源文件       |    无    |
|   `.i`   |   预处理后的文件   |   `-E`   |
|   `.s`   |    编译后的文件    |   `-S`   |
|   `.o`   | 汇编后的二进制文件 |   `-c`   |

>   第一步 : 对源文件进行预处理, 参数为`-E`

```shell
# 1. 预处理
$ gcc -E test.c -o test.i
```

>   第二步 : 对预处理后的程序进行编译, 参数为`-S`

```shell
# 2. 编译
$ gcc -S test.i -o test.s
```

>   第三步 : 对得到的汇编文件进行汇编, 参数为`-c`

```shell
# 3. 汇编
$ gcc -c test.s -o test.o
```

>   第四步 : 将得到的二进制文件和标准库进行链接, 得到可执行的二进制文件, 不需要参数

```shell
# 4. 链接
$ gcc test.o -o test
```



```shell
# 该命令是直接进行链接生成可执行程序, 前三步会自动进行
$ gcc test.c -o test
```



### 常用参数

|       参数       |                             作用                             |
| :--------------: | :----------------------------------------------------------: |
|       `-E`       |             只进行预处理，不进行编译、汇编和链接             |
|       `-S`       |           预处理并编译为汇编代码，不进行汇编和链接           |
|       `-c`       |           预处理、编译并汇编为目标文件，不进行链接           |
| `-o target_file` |      `source_file -o target_file`将源文件编译成目标文件      |
|       `-g`       |      生成调试信息, 使调试器能够查看源代码, 变量和调用栈      |
|     `-Wall`      |                       开启一组常用警告                       |
|       `-w`       |                        不生成警告信息                        |
|  `-I directory`  |                   指定引用头文件的搜索路径                   |
|     `-O[n]`      | `n`的取值范围：0~3。编译器的优化选项的4个级别，`-O0`不进行优化，通常用于调试，`-O1`为较少优化，`-O3`优化级别最高 |
| `-std=standard`  |                    指定语言规则`-std=c11`                    |



## 调试 - GDB

### 安装

```shell
$ sudo apt install gdb
```



### 调试准备

>   程序如果是为了调试而编译, 需要添加参数`-g`, 另外还有一些可选项, 在不影响程序行为的情况下关闭编译器的优化选项`-O0`, 打开常用警告`-Wall`

```shell
$ gcc test.c -o test -g -Wall -O0
```



### 启动和退出 gdb

>   gdb 启动后, 被调试的程序是没有执行的

```shell
# 在终端中执行如下命令
# gdb程序启动了, 但是可执行程序并没有执行
$ gdb filename
# 运行 run == r, 如果设置了断点会停在第一个断点的位置, 如果没有设置断点, 会执行完程序
(gdb) run
(gdb) r

# 运行 start, 启动程序, 会阻塞在 main 的第一行, 等待后续指令
(gdb) start

# 退出 quit == q
(gdb) quit
(gdb) q
```



|       命令       |            作用            |
| :--------------: | :------------------------: |
|   `run` / `r`    |        从头运行程序        |
|     `start`      | 运行并临时停在 `main` 开头 |
| `continue` / `c` | 继续运行到下一个断点或信号 |
|   `kill` / `k`   |     终止当前被调试进程     |
|   `quit` / `q`   |          退出 GDB          |



### 传参

```shell
# 启动时传参
$ gdb --args test hello 123
# 查看设置的命令行参数
(gdb) show args
Argument list to give program being debugged when it is started is "hello 123".

# 启动后, 程序启动前传参
(gdb) set args arg1 arg2 .... ...

# run 后直接传参数
(gdb) run hello 123
```



### 查看代码

>   查看代码的命令为`list` / `l`
>
>   ```shell
>   list / l [文件名:][行号 / 函数名]
>   ```
>
>   一个项目中一般有很多源文件, 默认情况下通过`list`查看到代码信息位于`main`对应的那个文件中, 如果不进行文件切换`main`函数所在的文件就是当前文件, 如果进行了切换, 切换到哪个文件, 哪个文件就是当前文件
>

-   当前文件


```shell
# 从第一行开始显示, 一次显示10行
(gdb) list 

# 列值这行号对应的上下文代码, 默认情况下只显示10行内容
(gdb) list 行号

# 显示这个函数的上下文内容, 默认显示10行
(gdb) list 函数名
```

-   切换文件

```shell
# 切换到指定的文件，并列出这行号对应的上下文代码, 默认情况下只显示10行内容
(gdb) l 文件名:行号

# 切换到指定的文件，并显示这个函数的上下文内容, 默认显示10行
(gdb) l 文件名:函数名

```

-   常见用法

```shell
# 下翻代码
(gdb) list
# 上翻代码
(gdb) list -
# 查看 20 行附近的代码
(gdb) list 20
# 查看 main 附近的代码
(gdb) list main
# 查看 test.c 文件 20 行附近的代码
(gdb) list test.c:20
# 查看 test.c 文件 Test 函数附近的代码
(gdb) list test.c:Test
```



### 断点

#### 设置断点

>   设置断点的命令为`break` / `b`
>
>   `tbreak` / `tb` 临时断点, 触发一次后自动删除
>
>   ```shell
>   break / b [文件名:][行号 / 函数名]
>   ```

-   设置普通断点到当前文件

```shell
# 在当前文件上设置断点
# break == b
(gdb) b 行号
(gdb) b 函数名
```

-   设置普通断点到目标文件文件

```shell
# 在目标文件上设置断点
(gdb) b 文件名:行号
(gdb) b 文件名:函数名
```

-   设置条件断点

```shell
# 必须要满足某个条件, 程序才会停在这个断点的位置上
# 通常情况下, 在循环中条件断点用的比较多
(gdb) b 行数 if 变量名==某个值
```



#### 查看断点

>   设置完断点后, 可以使用`info break` / `i b`查看断点信息 

```shell
# info == i
# 查看设置的断点信息
(gdb) i b   #info break

# 举例
(gdb) i b
Num     Type           Disp Enb Address            What
1       breakpoint     keep y   0x0000000000400cb5 in main() at test.cpp:12
2       breakpoint     keep y   0x0000000000400cbd in main() at test.cpp:13
3       breakpoint     keep y   0x0000000000400cec in main() at test.cpp:18
4       breakpoint     keep y   0x00000000004009a5 in insertionSort(int*, int) 
                                                   at insert.cpp:8
5       breakpoint     keep y   0x0000000000400cdd in main() at test.cpp:16
6       breakpoint     keep y   0x00000000004009e5 in insertionSort(int*, int) 
                                                   at insert.cpp:16
```

|     属性      |                             信息                             |
| :-----------: | :----------------------------------------------------------: |
|      Num      | 断点的编号, 删除断点和设置断点状态的时候都要用. 删除后该编号不会被重新使用 |
|     Type      | 表示这个断点是什么类型, `breakpoint`（普通断点）, `watchpoint`（监视点）, `catchpoint`（捕获点）等 |
| Disp(display) | 断点命中后的处理方式`keep`：触发后继续存在  `del`：触发一次后自动删除（一次性断点）  `dis`：触发后禁用 |
|  Enb(enable)  |             当前断点状态, `y`为可用, `n`为不可用             |
|    Address    |                      断点所在的内存地址                      |
|     What      |                        断点的位置描述                        |



#### 删除断点

>   删除断点的命令为 `delete` / `del` / `d`
>
>   删除断点的方式有两种: 删除指定断点或者删除一个连续的断点区间

```shell
(gdb) d 断点的编号1 [断点编号2 ...]

(gdb) d 1          # 删除第1个断点
(gdb) d 2 4 6      # 删除第2,4,6个断点

# 删除一个范围, 断点编号 num1 - numN 是一个连续区间
(gdb) d num1-numN
# 删除第1到第5个断点
(gdb) d 1-5
```



#### 忽略断点

>   可以使用`ignore`命令来忽略指定断点, 通常在调试循环时使用
>
>   ```shell
>   ignore N COUNT
>   ```

```shell
# 忽略 1 号断点 10 次
(gdb) ignore 1 10
```



#### 设置断点状态

>   如果某个断点只是临时不需要了, 我们可以将其设置为不可用状态, 设置命令为`disable 断点编号`, 当需要的时候再将其设置回可用状态, 设置命令为` enable 断点编号`
>

-   设置断点无效

```shell
# 让断点失效之后, gdb调试过程中程序是不会停在这个位置的
# disable == dis
# 设置某一个或者某几个断点无效
(gdb) dis 断点1的编号 [断点2的编号 ...]
# 设置第2, 第4 个断点无效
(gdb) dis 2 4


# 设置某个区间断点无效
(gdb) dis num1-numN
# 设置 第5,6,7,8 断点无效
(gdb) dis 5-8
```

-   让无效的断点生效

```shell
# enable == ena
# 设置某一个或者某几个断点有效
(gdb) ena 断点1的编号 [断点2的编号 ...]
# 设置第2, 第4个断点有效
(gdb) ena 2 4

# 设置某个区间断点有效
(gdb) ena num1-numN
# 设置第5,6,7个断点有效
(gdb) ena 5-7
```



### 调试命令

#### 继续执行

>   如果调试的程序被断点阻塞了又想让程序继续执行, 可以使用`continue` / `c`命令. 程序会运行到下一个断点

```shell
# continue == c
(gdb) continue
```



#### 手动打印信息

-   打印变量值

>   当程序被某个断点阻塞, 可以通过命令打印变量的名字, 类型, 数值

```shell
# 打印变量 print == p
(gdb) p 变量名

(gdb) p i
$5 = 3

# 如果想以特定类型格式输出
(gdb) p/fmt 变量名

# 16进制
(gdb) p/x i
$6 = 0x3
# 8进制
(gdb) p/o i
$7 = 03
```

| 格式化字符(/fmt) |          说明          |
| :--------------: | :--------------------: |
|       `/x`       |   以十六进制形式打印   |
|       `/o`       |    以八进制形式打印    |
|       `/d`       | 以有符号十进制形式打印 |
|       `/u`       | 以无符号十进制形式打印 |
|       `/t`       |    以二进制形式打印    |
|       `/f`       |    以浮点数形式打印    |
|       `/c`       |     以字符形式打印     |

-   打印变量类型

>   如果在调试过程中需要查看某个变量的类型, 可以使用命令`ptype`

```shell
(gdb) ptype 变量名

# 打印变量类型
(gdb) ptype i
type = int
(gdb) ptype array[i]
type = int
(gdb) ptype array
type = int [12]
```



#### 自动打印信息

-   设置变量名自动显示

```shell
# 在变量的有效取值范围内, 自动打印变量的值
(gdb) display 变量名

# 以指定的格式打印变量的值
(gdb) display/fmt 变量名
```

-   查看自动显示列表

```shell
# info == i
(gdb) info display
Auto-display expressions now in effect:
Num Enb Expression
1:   y  i
2:   y  array[i]
3:   y  /x array[i]
```



|    属性    |             信息             |
| :--------: | :--------------------------: |
|    Num     |      变量或表达式的编号      |
|    Enb     |  当前变量或表达式的激活状态  |
| Expression | 自动打印的变量或表达式的名字 |



-   取消自动显示

```shell
# 命令中的 num 是通过 info display 得到的编号, 编号可以是一个或者多个
(gdb) undisplay num1 [num2 ...]
(gdb) delete display num1 [num2 ...]
# 删除变量显示, 需要使用 info display 得到的变量/表达式编号
(gdb) undisplay 1 2

# num1 - numN 表示一个范围
(gdb) undisplay num1-numN
(gdb) delete display num1-numN
(gdb) undisplay 3-4
```



-   禁用或激活自动显示

```shell
# 禁用自动显示
(gdb) disable display num1 [num2 ...]
(gdb) disable display num1-numN

# 启用自动显示
(gdb) enable display num1 [num2 ...]
(gdb) enable display num1-numN
```



### 单步调试

>   当程序阻塞到某个断点上之后, 可以通过以下命令对程序进行单步调试

-   `step` / `s`

>   命令被执行一次代码被向下执行一行, 如果这一行是一个函数调用, 那么程序会进入到函数体内部

```shell
# 从当前代码行位置, 一次调试当前行下的每一行代码
# step == s
# 如果这一行是函数调用, 执行这个命令, 就可以进入到函数体的内部
(gdb) step
```



-   `finish`

>   通过`step`进入函数调用内部, 想要跳出函数体, 可以执行`finish`. 如果函数体内还有其他断点则无法跳出, 会先停在那个断点上

```shell
(gdb) finish
```



-   `next` / `n`

>   和`step`类似, 只是不会进入函数体内

```shell
# next == n
# 如果这一行是函数调用, 执行这个命令, 不会进入到函数体的内部
(gdb) next
```



-   `until`

>   可以直接跳出某个循环体. 条件为 : 1. 循环体内不能有其他断点 2. 必须在循环体开始 / 结束行执行

```shell
(gdb) until
```



### 设置变量值

>

```shell
# 可以在循环中使用, 直接设置循环因子的值
(gdb) set var 变量名=值
```
